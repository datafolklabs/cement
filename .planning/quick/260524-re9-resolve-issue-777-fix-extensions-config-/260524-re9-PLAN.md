# Plan 260524-re9: Fix extensions config-override to honor Meta.config_section

## Goal

Resolve GitHub issue #777 by fixing the `extensions` config-override block in
`_setup_config_handler` (cement/core/foundation.py:1489-1512) so it
reads/writes against `self._meta.config_section` instead of the bare
`label` local. This mirrors the parallel fix landed for `template_dirs` in
PR #760. The bug is silent in the common case (`Meta.config_section is None`
defaults to `Meta.label` at lines 1370-1371) but causes extensions defined
in configuration to be silently dropped whenever an app sets
`Meta.config_section` to a value other than `Meta.label`. Add a regression
test that exercises the divergent `config_section != label` case for both
the list and comma-separated-string input shapes ŌĆö matching the existing
two-branch coverage shape of `test_load_extensions_via_config`
(tests/core/test_foundation.py:469) so the 100% coverage gate stays green.
No public-API change, no BC break for the `config_section is None` path.
