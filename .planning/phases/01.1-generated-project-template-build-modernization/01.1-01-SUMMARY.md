### Verbatim file content (post-commit)

```toml
# Build-system declaration for pip's default PEP 517 build isolation.
# cement is required at build time because setup.py transitively imports
# it via {{ label }}/core/version.py. Project metadata stays in setup.py
# — this file declares ONLY the build system.

[build-system]
requires = ["setuptools", "wheel", "cement>={{ cement.version }}"]
build-backend = "setuptools.build_meta"
```

The literal substring `{{ cement.version }}` is present and will be
rendered by `cement/ext/ext_generate.py:_generate()` (which sets
`data['cement']['version'] = get_version()`) at `cement generate project`
time. With the current `VERSION = (3, 0, 15, 'final', 0)` in
`cement/core/backend.py`, the rendered line in a generated project's
pyproject.toml will be:

```toml
requires = ["setuptools", "wheel", "cement>=3.0.15"]
```

(Confirmed by reading the code; not yet confirmed by running the smoke
test in this worktree — see Acceptance Verification.)
