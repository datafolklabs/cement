<objective>
Migrate `cement generate project`'s template from setuptools+setup.py to
pdm-backend with a complete `[project]`-table-driven pyproject.toml. After
this plan lands, generated projects build cleanly under default PEP 517
isolation with no special pip flags — `pip install .` just works on Python
3.10–3.14 against a published cement.

Purpose: unblock Phase 2's CI-green prerequisite. The previous
strict-minimum patch (commit `8e7fba5e` adding a build-system stub +
commit `3d60764c` adding a `PIP_FIND_LINKS` band-aid to the smoke test)
was empirically insufficient because `setup.py` line 3 imports the
package being built (`from {{ label }}.core.version import get_version`)
which fails under PEP 517 build isolation regardless of
`[build-system].requires` configuration. This plan supersedes both prior
commits with a proper pdm-backend migration.

Output: 6 atomic conventional commits on the current branch
(`modernization-phase-1.1-project-templates`), all under wave 1, executed
sequentially, plus a 7th task that is verification-only (no commit).
The acceptance gate (`make cli-smoke-test` on the full 3.10–3.14 matrix)

states between tasks are intentionally inconsistent (e.g. the smoke-test
fix in task 1 expects the new template shape, but the template does not

rules we do NOT run the smoke test on intermediate states.
