## Decisions Made
- **Pin generated ruff/mypy to cement's line** (`ruff~=0.15.12`, `mypy~=1.20.2`) so the Plan 05 CI smoke-test is deterministic and enforces the same gate suite cement itself runs (D-04).
- **Ship `disable_error_code=[attr-defined]` in generated `[tool.mypy]`** — the generated app uses the Cement handler/Meta pattern where `self.app.args` is typed as the `ArgumentHandler` interface but resolves to the concrete argparse handler at runtime (`self.app.args.print_help()` is a documented, runtime-valid idiom used in 4 templates). cement's own mypy config disables the same code for the same reason. Chosen over inline `# type: ignore` noise in a teaching template.
- **Guard `app.last_rendered`/`output` with `assert ... is not None`** in the test template instead of altering `output.find(...)` assertion semantics — preserves behavior while satisfying mypy `strict_optional`.

**1. [Rule 3 - Blocking] Shipped ruff `I` (isort) rule flagged pre-existing template import ordering**
- **Found during:** End-to-end D-04 verification after Task 3
- **Issue:** The newly-shipped `extend-select` includes `I`; the template's import blocks (main.py, controllers/base.py, conftest.py, test_{{ label }}.py) were not isort-clean → `ruff check .` returned 4 I001 errors, breaking the "green out of the box" contract the plan's must_haves require.
- **Fix:** Reordered/blank-separated imports to isort canonical form (third-party group / blank / relative group, alphabetical; PEP8 two-blank-line separation before top-level defs).
- **Files modified:** main.py, controllers/base.py, conftest.py, test_{{ label }}.py
- **Verification:** `ruff check .` → "All checks passed!"
- **Committed in:** `d6c3c549`

**2. [Rule 3 - Blocking] mypy `strict_optional` errors on `app.last_rendered` unpacking**
- **Found during:** End-to-end D-04 verification
- **Issue:** `last_rendered` is typed `tuple[dict, str | None] | None`; unpacking it and calling `output.find(...)` triggered `[misc]` ("None object is not iterable") and `[union-attr]` errors under the shipped mypy config.
- **Fix:** Added `assert app.last_rendered is not None` and `assert output is not None` guards in the test template (behavior preserved — jinja2 always renders a string at runtime).
- **Files modified:** test_{{ label }}.py
- **Verification:** `mypy` → "Success: no issues found in 12 source files"
- **Committed in:** `d6c3c549`

**3. [Rule 3 - Blocking] mypy `attr-defined` on `self.app.args.print_help()`**
- **Found during:** End-to-end D-04 verification
- **Issue:** `self.app.args` is typed as the `ArgumentHandler` interface, which declares only `add_argument`/`parse` — not `print_help` (a concrete-handler method). Under the shipped mypy config this is an `[attr-defined]` error.
- **Fix:** Added `disable_error_code=["attr-defined"]` to the generated `[tool.mypy]` with an inline justification comment, mirroring cement's own mypy config rationale for the handler/Meta dynamic-attribute pattern.
- **Files modified:** pyproject.toml
- **Verification:** `mypy` exit 0
- **Committed in:** `d6c3c549`
