# Phase 05.3 Plan 06: Gap-Closure (todo ruff, false-green assert, changelog) Summary

Closed the three blocker gaps from 05.3-VERIFICATION.md with minimal correct
diffs: todo-tutorial `make comply` (ruff) is now green out of the box, the
canonical project test template asserts membership instead of the false-green
`str.find()` idiom, and the active 3.0.15 CHANGELOG section documents the 05.3
work — all verified against RENDERED template output, not the template source.

## What Was Built

- **Gap 1a** — isort-grouped the import blocks in the three Plan-04-modified todo
  template files (`todo/main.py`, `todo/controllers/base.py`,
  `todo/controllers/items.py`): stdlib / third-party / local groups separated by
  a single blank line. Ported byte-exact from `ruff check --fix --select I` run
  on a freshly generated todo-tutorial (no Jinja2 substitutions inside these
  import blocks, so the rendered fix equals the template edit).
- **Gap 1b** — scoped the todo-tutorial `[tool.ruff]` with
  `extend-exclude = ["tests"]` (mirroring the existing mypy `files = ["todo/"]`
  Option-B scope, with an explanatory comment), so the pre-existing
  `tests/conftest.py` lint bugs (F821, UP004) do not block comply-ruff.
  `tests/conftest.py` is unmodified.
- **Gap 1c** — added `test_generate_todo_ruff_clean(tmp)` to `tests/cli/test_main.py`:
  generates todo-tutorial in-process, then runs `subprocess.run(['ruff','check','.'],
  cwd=tmp.dir)` against the RENDERED output and asserts exit 0. This fills the
  D-07 hole where CI's cli-smoke-test todo block is build/install-only and never
  runs `make comply` on the generated todo.
- **Gap 2** — switched the canonical project test template's two false-green
  `assert output.find('Foo => ...')` assertions (str.find() returns -1, which is
  truthy) to membership `assert 'Foo => ...' in output`.
- **Gap 3** — added `[cli]`/`[dev]`-bucketed entries under the active
  `## 3.0.15 - DEVELOPMENT` section (Bugs / Refactoring / Misc) covering the 05.3
  packaging migration, gate config, typing work, the TodoError fix, the assertion
  fix, the CI smoke-test gate, and the new ruff guard.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Trailing blank line after todo/main.py import block**
- **Found during:** Task 1 verification (`ruff check .` still flagged I001 after
  the import reorder).
- **Issue:** ruff's isort (I001) normalizes `lines-after-imports`; the template
  kept two blank lines after the import block, so the gate stayed red even with
  correctly grouped imports.
- **Fix:** Removed one trailing blank line so the rendered output is byte-exact
  with `ruff --fix`. This is part of the I001 fix (import-block formatting), not
  a change to runtime code.
