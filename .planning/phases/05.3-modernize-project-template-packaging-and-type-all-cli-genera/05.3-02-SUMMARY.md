## Decisions Made
- **KEEP the script `VERSION` tuple** (D-08 discretion resolved): `(0, 0, 1, 'alpha', 0)` is the correct typed input to cement's `get_version(version: tuple[...]) -> str`, and the CI smoke-test asserts the resulting `myscript v0.0.1` banner. The script is intentionally single-file/lean; converting to file-source Shape B would add ceremony with no benefit.
- **Generated extension imports `App` via the top-level `from cement import App`** (not cement's internal `..core.foundation`). The generated extension is an external consumer of cement, so the public export is the correct import — this is why it differs from cement's own `ext_json.py` (which uses the relative internal path).
- **Preserve `self._parser.print_help()`** in the plugin controller: plugin controllers deliberately reach `self._parser` where project/script controllers use `self.app.args`. This is a documented behavior difference in the tutorial templates, not a bug to "fix."
