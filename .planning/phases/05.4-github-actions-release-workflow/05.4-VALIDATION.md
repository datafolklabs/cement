# Phase 05.4 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest + pytest-cov + coverage (repo standard); actionlint for workflow YAML if available |
| **Config file** | `pyproject.toml` `[tool.pytest.ini_options]` |
| **Quick run command** | `pdm run pytest --cov=cement.core tests/core` (core-only, service-free) |
| **Full suite command** | `make test` — needs redis(6379) + memcached(11211) + mailpit |
| **Estimated runtime** | ~60 seconds (quick), ~180 seconds (full) |

**Scope note:** This phase produces YAML workflows + `scripts/*` dev helpers, NOT
Python source in `cement/`. The 100% coverage gate applies to `cement/` only; new
`scripts/*.py` helpers must be ruff/mypy-clean but are outside the
coverage-measured surface (excluded from `[tool.pdm.build] includes`).

---

## Sampling Rate

- **After every task commit:** `actionlint` (if available) + `pdm run ruff check scripts/` + `pdm run mypy scripts/*.py` on any new Python helper; workflow YAML parses
- **After every plan wave:** Push branch → refactored `build_and_test.yml` (calling `gates.yml`) runs full PR CI including new Windows/macOS jobs
- **Before `/gsd-verify-work`:** A green `workflow_dispatch` dry-run of `release.yml` (preflight→gates→build→TestPyPI→smoke→summary)
- **Max feedback latency:** ~30 seconds locally (lint/type); CI-bound for workflow composition
