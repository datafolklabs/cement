<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git tag (author input) → release trigger | The tag name drives version, changelog, and downstream artifacts |
| runner → TestPyPI (OIDC) | Build artifacts are uploaded to an external index |
| build job → publish jobs (artifact handoff) | Isolating build from publish limits credential blast radius |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05.4-07 | Tampering | release trigger tag pattern | high | mitigate | Strict 3-part numeric tag guard `[0-9]+.[0-9]+.[0-9]+` (excludes moving tags 3/3.0, Pitfall 7) PLUS preflight semver + tag==version guard (D-02) as defense-in-depth |
| T-05.4-08 | Elevation of Privilege | default workflow permissions | high | mitigate | Top-level `permissions: contents: read`; only testpypi-publish elevates to `id-token: write`; no other job gets write scope in this plan |
| T-05.4-09 | Information Disclosure | build/test steps with publish creds | high | mitigate | Isolated `build` job (no publish creds); artifacts handed to publish via upload/download-artifact; OIDC short-lived token only in the publish job |
| T-05.4-10 | Tampering | TestPyPI dependency confusion | medium | mitigate | Smoke installs PLAIN zero-dep `cement==X.Y.Z` from TestPyPI only (Pitfall 2); no real-PyPI extra-index unless [cli] is smoked |
| T-05.4-SC | Tampering | GitHub Actions supply chain | high | mitigate | Exact-tag pins for gh-action-pypi-publish/upload-artifact/download-artifact; all first-party or PyPA per RESEARCH legitimacy audit |
</threat_model>
