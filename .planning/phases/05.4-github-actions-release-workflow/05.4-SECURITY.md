# Phase 05.4 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| TestPyPI index → runner | testpypi-smoke installs a package from an external index into the runner Python | package artifacts (public) |
| local files → release version-of-record | bump_dev_version rewrites the VERSION tuple that becomes the published version | version metadata |
| pull_request event → gate runners | Untrusted PR code runs in gate jobs (no secrets needed by gates) | untrusted code |
| reusable workflow → caller | gates.yml is invoked by both PR and release callers | CI control flow |
| git tag (author input) → release trigger | The tag name drives version, changelog, and downstream artifacts | release identity |
| runner → TestPyPI / PyPI (OIDC) | Build artifacts uploaded to external indexes via short-lived OIDC tokens | artifacts + OIDC claims |
| build job → publish jobs | Artifact handoff isolates build from publish credentials | dist artifacts |
| approval gate → publish side | GitHub Environment required-reviewer approval is the sole control before external publish | approval decision |
| runner → Docker Hub (token secret) | Access token authorizes image push | secret token |
| runner → stable/3.0.x + moving tags (contents:write) | Automated fast-forward push and moving-tag re-point | git refs |
| provider UIs → workflow run-time | Trusted publishers, environment reviewers, and secrets configured outside git authorize the run | provisioning state |
