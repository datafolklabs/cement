<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pull_request event → gate runners | Untrusted PR code runs in gate jobs (no secrets needed by gates) |
| reusable workflow → caller | gates.yml is invoked by both PR and release callers; a compromised gate could taint both |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05.4-04 | Elevation of Privilege | gates.yml reusable workflow | high | mitigate | gates.yml contains ONLY gate jobs — never a publish/OIDC job (RESEARCH anti-pattern); publish stays in release.yml. No `secrets: inherit` on the PR caller (gates need no secrets) |
| T-05.4-05 | Tampering | third-party action pins in gates.yml | high | mitigate | Every `uses:` pinned to an exact tag (CI-05); reuse in-repo pins verbatim; prefer first-party actions/ + community actions already vetted in RESEARCH legitimacy audit |
| T-05.4-06 | Elevation of Privilege | pull_request trigger | medium | mitigate | Gates trigger on `pull_request` (not `pull_request_target`) so untrusted PR code never runs with repo secrets |
| T-05.4-SC | Tampering | GitHub Actions supply chain | high | mitigate | Exact-tag pins + AUDIT-POINT discipline; RESEARCH Package Legitimacy Audit flagged no [ASSUMED]/[SUS]/[SLOP] actions |
</threat_model>
