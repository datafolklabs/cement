<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| approval gate → publish side | The GitHub Environment required-reviewer approval is the sole control before any external publish |
| runner → PyPI (OIDC) | Tokenless publish keyed on the environment claim |
| runner → Docker Hub (token secret) | Long-lived-ish access token authorizes image push |
| runner → stable/3.0.x (contents:write) | Automated fast-forward of a protected-ish branch |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05.4-11 | Elevation of Privilege | unauthorized/unreviewed publish | high | mitigate | `environment: release` required reviewers gate the single publish-pypi job — the one approval (D-04); all fan-out `needs: publish-pypi` so nothing publishes pre-approval |
| T-05.4-12 | Spoofing | PyPI OIDC trusted-publisher mismatch | high | mitigate | `environment: release` on the publish job emits the OIDC env claim; publish stays in top-level release.yml (never a reusable workflow) so the trusted-publisher workflow-filename match holds (Pitfall 1) |
| T-05.4-13 | Info Disclosure | Docker Hub token exposure | medium | mitigate | Token in repo/Environment secrets (never account password); scoped access token; used only in the docker job after approval; never echoed |
| T-05.4-14 | Tampering | stable/3.0.x forced update | medium | mitigate | Fast-forward-only push (`github.sha:refs/heads/stable/3.0.x`); non-descendant fails by design (Pitfall 6); job scoped to `contents: write` only |
| T-05.4-15 | Elevation of Privilege | over-broad GITHUB_TOKEN | high | mitigate | Least-privilege: each fan-out job elevates only its needed scope (contents / pull-requests / issues); id-token:write stays only on publish-pypi |
| T-05.4-SC | Tampering | GitHub Actions supply chain | high | mitigate | Exact-tag pins; prefer built-in `gh` for release (zero third-party); create-pull-request is the vetted community action per RESEARCH legitimacy audit |
</threat_model>
