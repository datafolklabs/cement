# Phase 05.1 Plan 02: type:choice picker + object-form boolean prompt Summary

**Completed the typed-variable surface on the unified resolver: `type: choice` now drives a numbered `shell.Prompt` picker emitting the chosen string at `data[name]`, and `type: boolean` gained the polymorphic object-form prompt `{text, accept, reject}` with a YAML bool-coercion load guard and scalar `extend.when` equality, all while the select fixtures migrated off the legacy bridge.**

## Accomplishments
- `type: choice` resolution: reuses the `SelectPrompt` numbered-picker construction (text + `options=labels` + `numbered=True` + `case_insensitive` + `default_label`), maps the chosen label -> option value via `values[labels.index(...)]`, and emits the chosen `str` at top-level `data[name]`. `options:` accepts the D-16 scalar list OR per-option `{value, prompt}` objects (both `str()`-coerced).
- Fail-fast `ValueError` choice validation (Q3): empty options, option object missing `value`, default not in option values, duplicate effective labels â€” all survive `python -O`.
- Polymorphic `type: boolean` object-form `prompt: {text, accept, reject}` (D-12): the author owns the full prompt text (no framework decoration); `accept:`/`reject:` are lowercased token lists mapping input -> bool; empty -> default; input matching neither asserts + aborts with `Invalid Response (must be one of accept/reject)` (D-14, mirrors `validate:`).
- YAML 1.1 bool-coercion load guard (Pitfall 1 / T-05.1-04): any `accept:`/`reject:` member that decoded as a Python bool raises `ValueError` telling the author to quote the token.
- `extend.when` SCALAR str()-coerced equality for choice/string (Q2): `when: flask` matches a `"flask"` option value; booleans still match by Python `==`. The migrated test16/test19 fire their per-option `ignore`/`variables` at the Plan-02 GREEN commit. List-form and string-regex `when` matching remain deferred to Plan 03.
- D-17 confirmed firing for `type: choice` (case:/validate: -> ValueError), covered by test37.
- Migrated the seven select fixtures (test16/19/20/21/26/28/30) to `type: choice` + `options:` + `extend:`; the now-dead legacy select-validation raises are pragma'd for the Plan-03 bridge removal.

## Decisions Made
- **Per-type `extend.when` dispatch** â€” booleans keep Python `==` (so `when: true` matches `True` and NOT the string `"True"`); choice/string use `str()`-coerced equality. See Deviations (none architectural).
