<objective>
Collapse the unreleased `features:` engine and the released `variables:` loop in
`cement/ext/ext_generate.py` into ONE ordered, recursive variable resolver. Add
the `type:` key (`string | boolean | choice`, default `string`), route every
variable through the top-level `data[name]` sink (delete the `data['features']`
namespace), and prompt all variables in a single declaration-order pass. This
plan delivers the engine core plus the `type: boolean` string-form prompt and
the three simplest bool fixtures so the 100% coverage gate holds at commit.

Purpose: This is the structural root of #782 — bug #4 (`{% if feature_x %}`
broken) is caused by features landing at `data['features'][name]` instead of
`data[name]`; the ordering complaint (#782 pt 1) is caused by the L261-264
feature pre-pass running before the L274 variable loop. Both are fixed here.
Output: refactored `ext_generate.py`, migrated test6/test7/test13 fixtures,
rewritten/new tests for the bool path. `extend.when` matching, `requires`,
`choice`, and the polymorphic object-form prompt land in plans 02-03.
</objective>
