- **Demo `.generate.yml` migrated** under `variables:` (no `features:`/`prompt_mode`/`enabled:`/`disabled:`): `project_name` (released string var, unchanged); `docker` -> `type: boolean default: true` with `extend: [{when: true, variables: [python_version]}, {when: false, ignore: [Dockerfile, .dockerignore]}]`; `docker_compose` -> `type: boolean default: true requires: [docker]` with `extend: [{when: false, ignore: [docker-compose]}]`; `web_framework` -> `type: choice default: "none"` with descriptive `{value, prompt}` options (D-16) + `extend:` keyed by `when: <value>` carrying per-branch `ignore` and the silent `framework_version` (`prompt: false`) for flask/fastapi.

- **All 5 cli templates verified:** grep confirms zero `features:`/`prompt_mode`/`enabled:`/`disabled:` in any of project/plugin/extension/script/todo-tutorial `.generate.yml`; EACH generates successfully under `--defaults` on the refactored engine and emits its expected stable file (project->`pyproject.toml`, plugin->`myplugin/__init__.py`, extension->`ext_myextension.py`, script->`myscript.py`, todo-tutorial->`Makefile`). ## Decisions Made
- **Where to demonstrate #782:** placed the `{% if %}` conditionals in `app.py` and the webapp `README.md` (files that always render) rather than inside `Dockerfile`/`docker-compose.yml`, which are themselves `ignore:`-suppressed when their feature is off -- an internal conditional there would be dead weight.
- **Interactive-only branches:** the docker-off and fastapi paths require interactive selection (the generate CLI exposes only `--defaults` headless dispatch, no per-variable flags). Their resolver behavior is already locked by the test suite (test9 requires-cascade, test16/test19 choice + silent var, test25 requires gate-fail); the demo README documents the expected output trees for those paths.
- **Task 2 produced no commit:** RESEARCH (Migration Surface) confirmed none of the 5 cli templates use features -- so D-19's "rewrite the cli templates" correctly reduces to "verify unchanged + still generate." Zero bytes changed, so there was nothing to commit; the verification is recorded here.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Removed a literal `{% if docker %}` from comment prose in app.py**
- **Found during:** Task 1 (first demo generate run)
- **Issue:** The first draft of `app.py` had a comment line reading
  `# docker is a top-level boolean: {% if docker %} works directly`. Jinja2
  parses `{% if %}` tokens even inside comment text, so this opened a second,
  unclosed `if` -> `TemplateSyntaxError: Unexpected end of template`.
- **Fix:** Reworded the comment to prose with no literal jinja tag
  (`# docker is a top-level boolean, so the conditional above works directly.`).
- **Files modified:** demo/generate-features/templates/generate/webapp/app.py
- **Verification:** per-file jinja `from_string` compile of all webapp templates
  now passes; `myapp.py generate webapp --defaults` succeeds.
- **Committed in:** `9d0545f9` (Task 1)
