# Phase 05.2 Plan 01: ext.argparse command self-meta accessor (#670) Summary

Additive read-only `ArgparseController._command_meta` property that hands a
running command its own `CommandMeta` (label, `parser_options['help']`, etc.)
via `self._command_meta`, returning `None` outside a dispatched command and
never altering the released `func()` dispatch signature.

## What Was Built

- **Task 1 â€” accessor** (`cement/ext/ext_argparse.py`, commit `38293f7c`):
  Added an `@property` `_command_meta` returning `"CommandMeta | None"`. It
  guards on `hasattr(self.app.pargs, '__dispatch__')` (returns `None`, never
  raises â€” D-03), derives `func_name` from `__dispatch__.split('.')[1]`, and
  resolves the meta off the class function object via
  `getattr(self.__class__, func_name).__cement_meta__` (class-level, matching
  the shared object whose `.controller` is mutated in `_collect_commands` â€”
  D-07 caveat). Carries the Cement-4 func-signature-injection deferral comment
  both at the property and at the `_dispatch` `__dispatch__` block (D-06/D-07).
  `func()` dispatch left untouched.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] mypy `no-any-return` on `__cement_meta__`**
- **Found during:** Task 1 verification (`pdm run mypy`).
- **Issue:** `return getattr(self.__class__, func_name).__cement_meta__` returned
  `Any` (dynamic attribute), tripping `error: Returning Any from function
  declared to return "CommandMeta | None"`.
- **Fix:** Bound the result to an explicit local
  `meta: CommandMeta = getattr(self.__class__, func_name).__cement_meta__` and
  `return meta` â€” the exact pattern `_collect_commands` already uses at
  `getattr(self.__class__, member).__cement_meta__`. The PATTERNS sketch used a
  bare `return getattr(...)`; the explicit-local form was required to hold the
  mypy gate. No behavior change.
- **Files modified:** `cement/ext/ext_argparse.py`
- **Commit:** `38293f7c`

**2. [Rule 1 - Bug] Updated `test_base_default` expected string**
- **Found during:** Task 2.
- **Issue:** To cover the None branch via the no-sub-command path, `Base._default`
  now returns a meta-aware string; the pre-existing `test_base_default` asserted
  the old literal `"Inside Base.default"`.
- **Fix:** Updated that assertion to the new
  `"Inside Base.default : meta > None"` value (which itself proves the None
  branch end-to-end).
- **Files modified:** `tests/ext/test_ext_argparse.py`
- **Commit:** `060a81d1`

## Naming Decision

`_command_meta` had **no collision** on `ArgparseController` (no existing
`@property` in the file; reserved attrs are `_parser`, `_sub_parsers`,
`_controllers`, `_controllers_map`, `_dispatch_option`, `_meta`). The D-01
proposed name shipped unchanged.

## Threat Flags

None. The accessor parses no new input â€” it reads already-parsed

T-05.2-02 DoS disposition (`mitigate`: must never raise on no-dispatch) is
satisfied by the `hasattr` guard and covered by
`test_command_meta_none_outside_dispatch`.
