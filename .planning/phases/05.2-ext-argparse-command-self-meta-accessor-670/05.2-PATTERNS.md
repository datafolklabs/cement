## Pattern Assignments

### `cement/ext/ext_argparse.py` (add read-only property to `ArgparseController`)

**Analog 1 — naming + property-shape precedent: `self._parser`** (`ext_argparse.py:336-337`)

The class already exposes framework-set, user-read attributes with a leading
underscore. `_parser` is the closest shape precedent (`_meta` comes from
`MetaMixin`, so it is not a literal `@property` in this class — `_parser` is the
better local model for adding a new accessor). These are set in `__init__`:

```python
def __init__(self, *args: Any, **kw: Any) -> None:
    super().__init__(*args, **kw)
    self.app: App = None  # type: ignore
    self._parser: ArgumentParser = None  # type: ignore
```

**Collision check (done — name is clear):** No `@property` exists anywhere in
this file. Grep of `ArgparseController` instance attrs shows the reserved
names are `_parser`, `_sub_parser_parents`, `_sub_parsers`, `_controllers`,
`_controllers_map`, `_dispatch_option`, plus `_meta` (from `MetaMixin`). **No
`_command_meta` / `_command` collision** — the D-01 proposed name `_command_meta`
is safe. (Note `_command` is the argparse `dest='command'` *pargs* key set in
`_get_subparser_options`, not an attribute on the controller — no clash with a
controller property named `_command_meta`.)

**Analog 2 — the resolution one-liner to centralize: `_dispatch()`**
(`ext_argparse.py:818-847`). The accessor reads the *same* `__dispatch__` this
block reads. Copy this resolution shape (the `hasattr` guard + `.split('.')`):

```python
if hasattr(self.app.pargs, '__dispatch__'):
    # if __dispatch__ is set that means that we have hit a sub-command
    # of a controller.
    contr_label = self.app.pargs.__dispatch__.split('.')[0]
    func_name = self.app.pargs.__dispatch__.split('.')[1]
else:
    # ... no sub-command branch ...

if contr_label == 'base':
    contr = self
else:
    contr = self._controllers_map[contr_label]

if func_name is None:
    pass
elif hasattr(contr, func_name):
    func = getattr(contr, func_name)
    return func()
```

**Pattern to copy for the accessor body:**
- Guard with `hasattr(self.app.pargs, '__dispatch__')` → return `None` when
  absent (D-03: outside a dispatched command, no raise).
- Derive `func_name = self.app.pargs.__dispatch__.split('.')[1]`.
- Resolve the `CommandMeta` off the class function object — mirror the existing
  meta lookup in `_collect_commands()` (Analog 3) using `getattr(...).__cement_meta__`.
- Return the raw `CommandMeta` dataclass (D-04 — no wrapper).

**Analog 3 — how to fetch `__cement_meta__` from the class function:**
`_collect_commands()` (`ext_argparse.py:704-710`) is the canonical lookup of the
`CommandMeta` dataclass off the *class-level* function object:

```python
for member in dir(self.__class__):
    if member.startswith('_'):
        continue
    elif hasattr(getattr(self, member), '__cement_meta__'):
        func: CommandMeta = getattr(self.__class__, member).__cement_meta__
        func.controller = self
        commands.append(func)
```

> **Class-level sharing caveat (D-07 / CONTEXT line 103-107):** `__cement_meta__`
> lives on the class function object and its `.controller` is mutated to `self`
> here. The accessor therefore returns a **shared, class-level** object, not a
> per-call copy. Use `getattr(self.__class__, func_name).__cement_meta__` (not
> `getattr(self, ...)`) to fetch the meta, matching this line exactly.

**Type / annotation conventions to copy** (from this file, `CLAUDE.md` Conventions):
- Return annotation must be explicit: `-> "CommandMeta | None"`.
- `CommandMeta` is already imported/defined in-module (`ext_argparse.py:108-116`)
  — no new import needed.
- Property docstring: one-line description + `Returns:` section (matches
  `parse()` docstring style at `ext_argparse.py:77-89`).

**Decorator pattern:** Use a standard `@property` returning the dataclass:

```python
@property
def _command_meta(self) -> "CommandMeta | None":
    """
    One-line description.

    Returns:
        CommandMeta | None: ...
    """
    if not hasattr(self.app.pargs, '__dispatch__'):
        return None
    func_name = self.app.pargs.__dispatch__.split('.')[1]
    if not hasattr(self.__class__, func_name):
        return None  # pragma path per planner's coverage call
    return getattr(self.__class__, func_name).__cement_meta__
```

(Above is the *shape* — planner finalizes the exact guard set and any
`# pragma: nocover` placement to hold the 100% gate; see Test analog for which
branches need coverage.)

**D-07 code comment:** Add a comment near the accessor (and/or `_dispatch` at
`~L843`) noting that Cement 4 may introduce opt-in func-signature injection
(`def cmd(self, func_name, func_meta)`) as the ergonomic default. Comment only,
no behavior. Use the framework's `# FIXME:` / plain-comment convention already
present in the file.

### `tests/ext/test_ext_argparse.py` (add coverage for the accessor)

**Analog — dispatch-a-command-and-assert: `test_base_cmd1` + `Base` fixture.**

The established pattern is: a fixture controller method returns a value derived
from controller state, then the test runs the app via the `ArgparseApp`
(`TestApp`) fixture with `argv=[...]` and asserts on `app.run()`'s return.

Fixture command (`test_ext_argparse.py:33-39`):

```python
class Base(ArgparseController):
    class Meta:
        label = 'base'
        arguments = [(['--foo'], dict(dest='foo'))]

    @expose()
    def cmd1(self):
        return "Inside Base.cmd1"
```

Test driver (`test_ext_argparse.py:268-271`):

```python
def test_base_cmd1():
    with ArgparseApp(argv=['cmd1']) as app:
        res = app.run()
        assert res == "Inside Base.cmd1"
```

The `ArgparseApp` fixture (`test_ext_argparse.py:236-247`) wires the controllers:

```python
class ArgparseApp(TestApp):
    class Meta:
        argument_handler = ArgparseArgumentHandler
        handlers = [Sixth, Base, Second, Third, Fourth, Fifth, Seventh]
```

**Coverage branches the planner must cover (per CONTEXT + accessor shape):**

1. **Dispatched-command branch** — add an exposed command (e.g. on `Base` or a
   new local fixture controller) whose body reads `self._command_meta` and
   returns/asserts on it, then drive it with `ArgparseApp(argv=[...])` and
   assert `app.run()`. This is the `test_base_cmd1` shape, with the assertion
   reaching into the returned `CommandMeta`.
2. **`parser_options['help']` read** — give the exposed command an
   `@ex(help='...')`, have the body return `self._command_meta.parser_options['help']`,
   assert the help string round-trips. (This is the exact #670 ergonomic call
   site — see `Second.cmd2` at `:52-62` for an `@expose(arguments=[...])` fixture
   that already inspects `self.app.pargs` inside the body; same shape, swap to
   `self._command_meta`.)
3. **None branch** — access the property when no command is dispatched (e.g.
   instantiate/resolve the controller and read `_command_meta` with no
   `__dispatch__` on `pargs`, or call from a context where `pargs` lacks
   `__dispatch__`). Assert `is None`. Mirror how `test_base_default`
   (`:259-265`) exercises the no-sub-command path.

**Conventions to copy:** `def test_<feature>()` naming; `with ArgparseApp(...)
as app:` context manager; `from cement.core.foundation import TestApp`;
assertions via bare `assert`. New fixture controllers go in the top-of-file
fixture block (`:21-233`) and must be registered in `ArgparseApp.Meta.handlers`
(`:239-247`) if used.
