<objective>
Add a single additive, read-only `_command_meta` property to `ArgparseController`
in `cement/ext/ext_argparse.py` that returns the currently-dispatched command's
`CommandMeta` dataclass (so a command can read its own `@ex`/`@expose` decorator
meta from inside its body), returning `None` when no command is dispatched.
Resolves #670 without touching the released `func()` dispatch signature.

Purpose: Replace the brittle, repeat-yourself
`getattr(self, self.app.pargs.__dispatch__.split('.')[1]).__cement_meta__` dance
(now broken in the wild since `CommandMeta` became a dataclass) with the
one-liner `self._command_meta`, while preserving zero public-API breakage on the
3.0.x track (D-01..D-07).

Output: a documented property on `ArgparseController`, tests holding the 100%
coverage / ruff / mypy gates, and a CHANGELOG Features entry.
</objective>
